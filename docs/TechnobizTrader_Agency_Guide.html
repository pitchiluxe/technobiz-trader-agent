1. What Is TechnobizTrader?

TechnobizTrader is an autonomous AI trading agency composed of three specialised agents that work in a strict sequential pipeline. Each agent has one job, one input, and one output. They never skip steps or bypass each other's validation.

The strategy is ICT (Inner Circle Trading) — a methodology developed by Michael S. that identifies institutional order-flow through four specific market events before entering any trade.

The Four Mandatory ICT Elements

Analyse-Master requires ALL four of the following before generating any signal:

#ICT ElementWhat It MeansWhat the Code Looks For
1 Liquidity Sweep Retail stop-orders are triggered beyond a swing level, then price reverses — smart money collected liquidity. A wick that breaks a key swing high/low by ≥ 2 pips but the candle closes back inside the level.
2 Break of Structure (BoS) Price closes beyond the prior swing point, confirming a directional shift or continuation. Most-recent close above the 2nd-to-last swing high (bullish) or below the 2nd-to-last swing low (bearish).
3 Imbalance / Order Block An unfilled price gap (Fair Value Gap) or the last opposing candle before an impulse — institutional entry zone. 3-candle FVG: prev.high < next.low. OB: last bearish candle before a 3-candle bullish impulse (or vice versa).
4 Pullback Entry Price retraces into the Order Block / FVG zone, giving a low-risk entry aligned with institutional bias. Current candle's wick touching or entering the OB/FVG zone (within ± 3 pip buffer).

2. Step-by-Step: One Trading Cycle (≈ 15 minutes)

The system runs a trading cycle every 15 minutes. Here is every internal step in the exact order the code executes them.

Phase 0 — System Initialisation (one-time at startup)

0

Load Configuration

  • Read .env file → config/settings.py singleton
  • Load MT5 credentials: account, password, server
  • Set risk params: MAX_RISK=2%, MAX_TRADES=3, MIN_CONFIDENCE=75%
  • Connect to SQLite / PostgreSQL database
A

MT5 Terminal Connection

  • Call mt5.initialize() to start the terminal bridge
  • Call mt5.login(account, password, server)
  • Verify account balance and currency via mt5.account_info()
  • If connection fails → abort startup, log error
B

Instantiate All Three Agents

  • TrendMaster(verbose=False)
  • AnalyseMaster(verbose=False)
  • TraderMaster(verbose=False)
  • TradingWorkflow orchestrator wraps all three

Phase 1 — Data Fetch (every cycle)

1

Pull Live OHLC Data from MT5

  • Daily candles: mt5.copy_rates_from_pos("EURUSD", TIMEFRAME_D1, 0, 200)
  • 4-Hour candles: mt5.copy_rates_from_pos("EURUSD", TIMEFRAME_H4, 0, 200)
  • 1-Hour candles: mt5.copy_rates_from_pos("EURUSD", TIMEFRAME_H1, 0, 200)
  • Each raw row is converted to OHLCData(timestamp, open, high, low, close, volume)
  • Result is stored as market_data = {"daily": [...], "4h": [...], "1h": [...]}

Phase 2 — TREND-MASTER Analysis (Step 1 of pipeline)

2

Multi-Timeframe Swing Detection

  • Run _find_swing_highs(candles, window=5) on Daily, 4H, 1H
  • Run _find_swing_lows(candles, window=5) on all three timeframes
  • Rolling window: if candle[i].high beats all neighbours ±5 candles → swing high
3

Market Structure Classification

  • Check last 3 swing highs and lows per timeframe
  • Highs and lows both ascending → BULLISH
  • Highs and lows both descending → BEARISH
  • Mixed or insufficient swings → NEUTRAL
  • Final bias = majority vote across all 3 timeframes
4

Support / Resistance Level Validation

  • Collect all swing highs/lows from Daily + 4H (higher-weight data)
  • For each candidate level: count how many candles touched it (within 5 pip tolerance)
  • Only include levels with ≥ 2 historical touches
  • Deduplicate levels that are within 15 pips of each other
5

ATR Volatility Check & Confidence Scoring

  • ATR(14) = average true range of last 14 one-hour candles
  • If current ATR > long-term ATR × 1.5 → risk level = HIGH (−10% confidence)
  • Base confidence: 60 + 10 (all 3 TFs agree) + 10 (≥3 S/R levels) − 10 (high vol)
  • Output: TrendReport with bias, confidence, S/R levels, liquidity zones
Stop condition: If Trend-Master returns None (insufficient candles, NEUTRAL bias, or HIGH risk), the cycle ends here. No signal is generated, no trade is placed.

3. Step-by-Step Continued — Analyse-Master & Trader-Master

Phase 3 — ANALYSE-MASTER ICT Detection (Step 2 of pipeline)

6

ICT Element 1 — Liquidity Sweep Detection

  • Scan last 5 one-hour candles
  • Bullish: any candle's low dips > 2 pips below the lowest swing low, but closes above it
  • Bearish: any candle's high pokes > 2 pips above the highest swing high, but closes below it
  • Result: sweep_ok = True/False, stores the swept price level
7

ICT Element 2 — Break of Structure

  • Bullish BoS: latest close > 2nd-to-last swing high → confirms upward momentum
  • Bearish BoS: latest close < 2nd-to-last swing low → confirms downward momentum
  • Must align with the bias from Trend-Master
  • Result: bos_ok = True/False
8

ICT Element 3 — Imbalance / Order Block

  • Fair Value Gap (FVG): search last 20 candles for a 3-candle gap (prev.high < next.low for bullish)
  • Order Block (OB): last opposing candle before a 3-candle impulse (e.g. last bearish candle before 3 bullish candles with total body ≥ 5 pips)
  • OB is preferred over FVG; FVG used as fallback
  • Zone is recorded as (ob_bottom, ob_top)
  • Result: imbalance_ok = True/False
9

ICT Element 4 — Pullback Entry

  • Check if the current (latest) candle has a wick touching or inside the OB/FVG zone
  • Bullish: current low ≤ ob_top + 3 pip buffer AND ≥ ob_bottom − 3 pip buffer
  • Bearish: current high ≥ ob_bottom − 3 pip buffer AND ≤ ob_top + 3 pip buffer
  • Result: pullback_ok = True/False
10

Risk/Reward Calculation & Signal Construction

  • Entry = midpoint of OB zone: (ob_bottom + ob_top) / 2
  • Stop Loss = OB edge + 3 pip buffer (beyond the zone)
  • TP1 = entry ± 1× risk  |  TP2 = entry ± 2× risk  |  TP3 = entry ± 3× risk
  • R:R ratio calculated on TP2 — must be ≥ 2.0
  • Confidence score = 50 + (15 × confirmed elements) + 5 (if R:R ≥ 3) × trend_confidence%
  • Kill zone: now → now + 30 minutes
  • Output: TradeSignal or None if any check fails

Phase 4 — TRADER-MASTER Execution (Step 3 of pipeline)

11

Pre-Execution Guardrail Checks

  • Confidence ≥ 75% → else REJECT
  • R:R ≥ 2.0 → else REJECT
  • Active open trades < 3 → else REJECT
  • Daily drawdown < 5% → else PAUSE
  • Kill zone not expired → else REJECT
  • 3 consecutive losses → ALLOW but halve lot size
12

Dynamic Position Sizing (2% Risk Rule)

  • Formula:
    risk_pips = |entry − stop_loss| ÷ 0.0001
    max_risk_$ = account_balance × 0.02
    lots = max_risk_$ ÷ (risk_pips × $10/pip)
  • Clamped to 0.01 (min) and 10.0 (max) lots
  • Example: $10 000 balance, 20-pip SL → max risk $200, lots = 200/(20×10) = 1.0 lot
13

Order Placement & Record Creation

  • Generate unique signal ID: SIG-{8-char UUID hex}
  • Place limit order via mt5.order_send() with SL and TP1
  • Status set to PENDING until filled
  • Create ExecutionRecord with all three TP levels logged
  • Record appended to TradingWorkflow.execution_records
  • Full row saved to TradeExecution database table

6. Risk Management Guardrails

These rules are hard-coded into the agents. They cannot be overridden by any signal, regardless of how high the confidence is.

🛡️ Per-Trade Rules

  • Maximum risk per trade: 2% of account balance
  • Minimum R:R ratio: 1:2 (TP2 must be ≥ 2× the risk)
  • Hard stop loss is non-negotiable — always set on order
  • Limit order entry only (market order as fallback for high urgency)
  • Entry timeout: 5 minutes (unfilled limit orders cancelled)

📊 Portfolio Rules

  • Maximum 3 concurrent open trades
  • Daily drawdown cap: 5% → all trading paused
  • 3 consecutive losses → position size halved for next 5 trades
  • No averaging down, no martingale, no pyramiding
  • 15-minute minimum between signals

📡 Signal Quality Rules

  • Minimum signal confidence: 75%
  • All 4 ICT elements must be confirmed simultaneously
  • Signal expires after 30 minutes (kill zone end)
  • No signals when ATR > average × 1.5 (extreme volatility)
  • NEUTRAL bias → no signal generated

⚙️ System Safeguards

  • MT5 connection verified before every data fetch
  • Any agent returning None stops the pipeline immediately
  • All executions written to database with full audit trail
  • Unique signal IDs: SIG-{uuid8} on every trade
  • Full exception catch-all in workflow orchestrator

10. Connecting the Agent to MetaTrader 5 (MQL5)

The agent communicates with MT5 through the official MetaTrader5 Python bridge. The bridge runs on the same Windows machine as the MT5 terminal — the Python process talks to the terminal over a local named pipe. No MQL5 script is needed.

Platform requirement: The MetaTrader5 Python package only works on Windows. For Linux/Mac hosting, you must run it inside a Windows VM or Windows container (see Section 11 for VPS options).

Step-by-Step: MT5 Connection Setup

1

Download & Install MetaTrader 5 Terminal

  • Go to metatrader5.com or your broker's download page
  • Install the MT5 terminal (Windows .exe installer)
  • Open the terminal and log in to your live or demo account
  • Go to Tools → Options → Expert Advisors and tick:
    ✅ Allow automated trading
    ✅ Allow DLL imports
    ✅ Allow WebRequests for listed URL
  • Leave the MT5 terminal open and logged in — the Python bridge needs it running
2

Install the Python MetaTrader5 Package

  • Open PowerShell or Command Prompt in your project folder
  • Run: pip install MetaTrader5
  • Verify install: python -c "import MetaTrader5 as mt5; print(mt5.__version__)"
  • Expected output: a version number like 5.0.37
  • If you get an error, ensure you are on Python 3.10+ on Windows
3

Configure Your Credentials in .env

  • Copy .env.template → .env in the project root
  • Fill in the MT5 section:
# ── MetaTrader 5 Credentials ────────────────────────── MT5_ACCOUNT=12345678 # Your MT5 account number (integer) MT5_PASSWORD=YourPassword # Your MT5 account password MT5_SERVER=ICMarkets-Demo # Exact server name (copy from MT5 Login screen) # ── Trading Parameters ──────────────────────────────── MT5_SYMBOL=EURUSD # Symbol to trade MAX_RISK_PER_TRADE=0.02 # 2% account risk per trade MAX_CONCURRENT_TRADES=3 # Max open positions MIN_CONFIDENCE_THRESHOLD=75 # Minimum signal confidence % # ── Database ────────────────────────────────────────── DATABASE_URL=sqlite:///trading.db # SQLite (dev) or postgresql://... # ── Azure AI Foundry (optional — enhances signal text) ─ FOUNDRY_PROJECT_ENDPOINT=https://your-project.openai.azure.com/ FOUNDRY_MODEL_DEPLOYMENT_NAME=gpt-4o
4

Find Your Exact Server Name

  • In MT5 terminal: top menu → File → Open Account
  • The server name is shown next to your account, e.g. ICMarkets-Demo02
  • Alternatively: bottom status bar of MT5 shows the server name
  • The name must be exact (case-sensitive) — typos cause silent login failure
5

Test the Connection Before First Run

  • Run the connection test script from the project root:

Enable the Symbol in MT5 Market Watch

  • In MT5: right-click the Market Watch panel → Show All
  • Find your symbol (e.g. EURUSD) and ensure it is checked/visible
  • If mt5.copy_rates_from_pos("EURUSD", ...) returns empty, the symbol is hidden
  • Also ensure Auto-Trading button (top toolbar) is green/enabled

12. Running 24/7 — Fully Automated Operation

Three approaches are provided — choose the one matching your setup.

Requirement for all methods: The MetaTrader 5 terminal must remain open and logged in on the same Windows machine at all times. MT5 does not support headless operation — it must have a visible (or minimised) window.

Approach A — Add the 15-Minute Loop to main.py (simplest)

Replace the single-cycle call in main.py with this continuous loop:

# main.py — continuous loop (add/replace the existing main() body) import asyncio, logging from agents.workflow import TradingWorkflow from market_data.mt5_provider import MT5Provider from config.settings import settings from database.db_manager import db_manager logger = logging.getLogger(__name__) CYCLE_INTERVAL_SECONDS = 900 # 15 minutes async def run_forever(): db_manager.create_tables() provider = MT5Provider(settings.MT5_ACCOUNT, settings.MT5_PASSWORD, settings.MT5_SERVER) await provider.connect() workflow = TradingWorkflow(verbose=settings.DEBUG) while True: try: # Fetch live candles market_data = { "daily": await provider.get_candles(settings.MT5_SYMBOL, "D", 200), "4h": await provider.get_candles(settings.MT5_SYMBOL, "4H", 200), "1h": await provider.get_candles(settings.MT5_SYMBOL, "1H", 200), } result = await workflow.execute_trading_cycle(market_data) if result: logger.info(f"Cycle complete — Trade placed: {result.signal_id}") else: logger.info("Cycle complete — No signal") except Exception as e: logger.error(f"Cycle error: {e}", exc_info=True) # log, don't crash # Wait 15 minutes before next cycle logger.info(f"Sleeping {CYCLE_INTERVAL_SECONDS}s until next cycle...") await asyncio.sleep(CYCLE_INTERVAL_SECONDS) if __name__ == "__main__": asyncio.run(run_forever())

Approach B — Windows Service with NSSM (recommended for production)

NSSM (Non-Sucking Service Manager) wraps your Python script as a proper Windows Service that starts automatically on boot, restarts on crash, and runs without a logged-in user session.

1

Install NSSM

  • Download from nssm.cc → extract to C:\nssm\
  • Add C:\nssm\win64\ to your Windows PATH
  • Or run commands with the full path: C:\nssm\win64\nssm.exe ...
2

Create the Windows Service

# Run PowerShell as Administrator nssm install TechnobizTrader # In the NSSM GUI that opens: Path: G:\SSD_Files\Trading_Agent\.venv\Scripts\python.exe Startup dir: G:\SSD_Files\Trading_Agent Arguments: main.py # Click Install Service
3

Configure Restart & Log Output

# Set log file for stdout nssm set TechnobizTrader AppStdout G:\SSD_Files\Trading_Agent\logs\service_out.log nssm set TechnobizTrader AppStderr G:\SSD_Files\Trading_Agent\logs\service_err.log # Auto-restart if it crashes (restart after 5 seconds) nssm set TechnobizTrader AppRestartDelay 5000 # Start the service nssm start TechnobizTrader # Check status nssm status TechnobizTrader # Expected: SERVICE_RUNNING # Stop / restart commands nssm stop TechnobizTrader nssm restart TechnobizTrader
4

Set MT5 to Auto-Start on Boot

  • Press Win + R → type shell:startup → press Enter
  • Create a shortcut to your MT5 terminal EXE in that folder
  • MT5 will now launch automatically when Windows boots
  • Configure MT5 to auto-login: File → Login to Trade Account → check Save password
  • Set Windows to auto-login on boot: netplwiz → untick password requirement

Approach C — Windows Task Scheduler (no install needed)

1

Open Task Scheduler

  • Press Win + S → search Task Scheduler → open it
  • Right-click Task Scheduler Library → Create Basic Task
  • Name: TechnobizTrader_24x7
2

Configure Trigger & Action

  • Trigger: When the computer starts
  • Action: Start a program
  • Program: G:\SSD_Files\Trading_Agent\.venv\Scripts\python.exe
  • Arguments: main.py
  • Start in: G:\SSD_Files\Trading_Agent
  • Security options: Run whether user is logged on or not → requires your Windows password
  • Tick Run with highest privileges
